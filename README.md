# LEET 기출 학습

LEET 추리논증과 논술 문제를 파트별로 학습하고, 정답·구조화된 해설·논술 공략법과 모범답안을 확인하는 정적 웹앱입니다.

## 구성

- 한 문항 단위의 앞면/해설면 플립 인터페이스
- 추리논증·논술 파트 분류 탭과 파트별 독립 문항 위치 저장
- 데스크톱 2단 구성과 모바일 통합형 문항 카드·전체 세로 스크롤
- 모바일에서 긴 문제 질문(발문)을 줄임 없이 여러 줄로 표시
- 미응답 상태의 해설 열람과 선지 재클릭 선택 해제
- 연도 선택, 문항 바로가기, 로컬 진도 저장
- 핵심 원리·단계별 풀이·주의할 함정과 논증 흐름·비교표·계산식 등 HTML 해설 시각화
- 논술 원문 조건·사례·의견 보존, 채점 포인트·시간 배분·단계별 공략·모범답안 3종
- 빌드 과정과 외부 런타임 의존성이 없는 GitHub Pages용 정적 파일

## 로컬 실행

ES 모듈을 사용하므로 파일을 직접 열지 말고 간단한 정적 서버로 실행합니다.

```bash
python3 -m http.server 4174
```

그다음 `http://localhost:4174`를 엽니다.

## 데이터 검증

```bash
node scripts/validate-data.mjs
node scripts/validate-essay.mjs
```

추리논증은 문항 번호, 선택지 수, 정답 범위, 공식 정답표, 조합형 선지와 해설의 O/X 일치 여부를 검사합니다. 논술은 원문 구역, 의견 10개, 모범답안 3종, 답안별 900~1200자와 의견 4개 이상 활용 여부를 검사합니다.

## 새 학년도 추가

1. `data/2027.js`를 참고해 `data/2028.js` 같은 새 파일을 만듭니다.
2. 각 문항에 `id`, `type`, `stem`, `sections`, `statements`, `choices`, `answer`, `explanation`을 작성합니다.
3. 같은 학년도의 `data/연도-details.js`에 문항별 `principle`, `steps`, `trap` 상세 해설을 작성합니다.
4. `data/index.js`에서 새 파일을 가져와 `exams` 배열에 추가합니다.
5. 필요하면 검증 스크립트에 해당 학년도 공식 정답표를 추가합니다.

`data/helpers.js`의 `section`, `table`, `flow`, `visualTable`, `equations`를 이용하면 문제 표와 해설 시각화를 같은 형식으로 추가할 수 있습니다.

## 새 논술 학년도 추가

1. `data/2027-essay.js`를 참고해 `data/2028-essay.js` 같은 파일을 만듭니다.
2. 문제 원문은 `conditions`, `case`, `opinions` 종류의 구역으로 나누되 문장과 문단은 그대로 전사합니다.
3. 각 문항에 출제 의도, 시간 배분, 단계별 공략, 답안 뼈대, 채점 포인트, 모범답안 3종을 작성합니다.
4. `data/index.js`의 논술 파트 `exams`에 새 학년도를 추가합니다.
5. 논술 검증 스크립트에 학년도별 문항 수와 조건을 추가합니다.

문제 영역은 원문을 전사하며, 문장·문단을 임의로 표나 요약문으로 바꾸지 않습니다. 표와 논증 도식은 원문에 실제로 존재할 때만 문제 영역에 사용합니다. 재구성한 비교표·흐름도는 해설 영역에만 둡니다.

원문에서 밑줄 친 범위는 `[[u]]밑줄 범위[[/u]]`로 감싸서 그대로 보존합니다. `㉠`, `㉡`, `ⓐ` 같은 참조 표식도 제목이나 메타데이터로 옮기지 말고 원문에서 인쇄된 위치에 입력합니다.

## 출처

문제 저작권은 법학전문대학원협의회에 있습니다. 이 저장소는 학습 편의를 위해 사용자가 제공한 2027학년도 추리논증 자료를 HTML 데이터로 구성한 비공식 학습 도구입니다.
